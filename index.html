3. 运行步骤

  1. 解析 part-00000

    跑 cc-chinese/1_parse.py，把每行 JSON-ish 记录里的 html 字段写到独立文件。

    cd /ytech_m2v8_hdd/workspace/kling_mm/zhengyuanhong
ln -sfn /Users/amjstj/zhengyuanhong/interleave/part-00000 \
       data/cc-chinese/part-00000
python3 interleave/cc-chinese/1_parse.py

    输出

    • outputs/cc-chinese/raw/html/{1..4975}.html
    • outputs/cc-chinese/raw/records.jsonl — {img_num, html_path} 每行一条
    • outputs/cc-chinese/raw/html_files.txt — html 路径清单（供后续流水线消费）

    耗时约 4 秒（多进程，25 batch）。

  2. Clone OBELICS（走代理）

    这台机器直连 github.com 超时；用公司 squid 代理才能拉到。

    export http_proxy=http://oversea-squid2.ko.txyun:11080
export https_proxy=http://oversea-squid2.ko.txyun:11080
export no_proxy=localhost,127.0.0.1,corp.kuaishou.com

git clone https://github.com/huggingface/OBELICS.git \
  /ytech_m2v8_hdd/workspace/kling_mm/zhengyuanhong/interleave/OBELICS

  3. 安装依赖

    OBELICS 的 requirements.txt 全部装上后，kenlm 必须从 github 源码装，仍然走代理。

    pip install selectolax bs4 warcio pathos pyyaml datasets img2dataset \
            emoji==1.6.1 fasttext sentencepiece

# 仅 kenlm 需要走代理
http_proxy=$http_proxy https_proxy=$https_proxy \
  pip install https://github.com/kpu/kenlm/archive/master.zip
    注意：这台机器的 pip 默认指向 pypi.corp.kuaishou.com，正常包不需要代理；只有 github 源（kenlm）才需要。

  4. 写中文友好的过滤配置

    OBELICS 默认配置面向英文 CommonCrawl —— stopword/common-word/perplexity/lang-id/flagged-word 这些过滤项都用英文词表与英文 KenLM。直接套用会把中文文档全部过滤掉。

    新增 obelics/configs/config_filter_web_documents_zh.yaml，关掉这些英文专用检查，保留结构性检查（图片尺寸/比例/字符重复/特殊字符比/图片数量等）。关键差异：

    cond_check_stopword_ratio_node_level: True   # 英文版
cond_check_stopword_ratio_node_level: False  # 中文版
cond_check_common_word_ratio_doc_level: True
cond_check_common_word_ratio_doc_level: False
cond_check_lang_id_doc_level: True
cond_check_lang_id_doc_level: False
cond_check_perplexity_score_doc_level: True
cond_check_perplexity_score_doc_level: False

  5. 写驱动 run_cc_chinese.py

    OBELICS 的 build 脚本（build_obelics/02_*.py、05_*.py）依赖 S3、SLURM-shape 的输入，不能直接跑。我写了一个本地驱动复用 OBELICS 的 processors：

    1. 读 html_files.txt，取前 --limit 条 → 构造 HuggingFace Dataset（html + url 列，url 从 <link rel="canonical"> 抽取）
    2. 跑 DOMTreeSimplificator + PreExtractionSimplificator → texts / images / metadata 三列
    3. 抽出所有 image URL → 调 img2dataset 下载（webdataset .tar 格式）
    4. 把图片字节 bind 回每条记录 → images_urls 列保留原 URL，images 列是 PIL bytes
    5. 跑 WebDocumentFilteringNodeLevel + WebDocumentFilteringDocLevel
    6. 结果保存为 HF dataset + jsonl

  6. 执行

    export http_proxy=... https_proxy=...   # img2dataset 要联外网

cd /ytech_m2v8_hdd/workspace/kling_mm/zhengyuanhong/interleave/OBELICS
python3 run_cc_chinese.py \
  --limit 50 --num-proc 4 \
  --download-images --thread-count 16

7. 遇到的坑与解法

① 网络不通

本机直连 github.com / huggingface.co / pypi.org 都超时（无 DNS 出口）。解法：用公司 squid 代理 http://oversea-squid2.ko.txyun:11080。pip 走公司内 PyPI 镜像不用代理，只有直连 github 的依赖（kenlm zip、git clone）才需要。

② OBELICS 没在 PyPI 上

无法 pip install obelics，必须 clone + 用 obelics.processors 模块。

③ build_obelics/* 脚本依赖 S3 / 200-shard 集群环境

不能直接跑。解法：自己写 run_cc_chinese.py，复用 processors，去掉 S3/分片逻辑。

④ 默认过滤配置全是英文导向

stopword/common-word/perplexity/lang-id 用英文词表与英文 KenLM。解法：关掉这些英文专用项，保留结构性过滤。

⑤ 类名错

obelics.processors 暴露的是 CommonCrawlWebDocumentExtractor，不是 WebDocumentExtractor。修了一行 import alias。

⑥ 不下载图片时过滤器报 'str' object has no attribute 'size'

OBELICS 的 node-level 过滤在 if image is not None: 分支里无条件访问 image.size。它假设 images 列已经被 urls_to_images 替换成 PIL 对象，而 URL 字符串也是 truthy。解法：要么真下载（首选），要么把 URL 临时替换成 None 再跑 node-level；driver 里两条路径都支持。

⑦ html_to_web_documents 会移除 url 列

它用 dataset.map(..., remove_columns=dataset.column_names)。解法：在 map 前 stash url，map 后 add_column 加回。
